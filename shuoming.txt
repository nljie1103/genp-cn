这是 GenP (Generic Patcher) v3.8.0，一个专门用于破解 Adobe Creative Cloud 系列软件（Photoshop、Illustrator、Premiere Pro、After Effects、Acrobat 等）的工具。它通过在目标 Adobe 二进制文件中搜索特定的十六进制模式，将其替换为修改后的字节，从而绕过许可证验证。

二、使用的语言和技术栈
组件	语言/技术	说明
GenP-3.8.0.au3	AutoIt v3	主程序，用 AutoIt 脚本语言编写。AutoIt 是一种 Windows 自动化/GUI 脚本语言，语法类似 BASIC
build.ps1	PowerShell	自动化构建脚本
run_build.bat	Batch	以管理员权限启动构建流程的启动器
patch_wintrust.ps1	PowerShell	补丁 wintrust.dll 的脚本
config.ini	INI 配置	定义目标文件列表和十六进制补丁模式
三、编译/构建流程
构建过程在 build.ps1 中定义，步骤如下：

必须以管理员身份运行 run_build.bat
脚本自动从官网下载 AutoIt v3 Portable (17MB) 和 SciTE4AutoIt3 Portable (7MB)
解压 UPX 目录中的 upx-5.0.1-win64.zip（加壳压缩工具），拷贝 upx.exe 到 GenP 目录
运行 patch_wintrust.ps1，对 wintrust.dll 进行二进制补丁（修改偏移 0x1C86-0x1C87 处的字节为 0x33 0xC0，即 xor eax, eax，让函数返回 0）
用 AutoIt3Wrapper 将 .au3 脚本编译为独立的 .exe 可执行文件（同时使用 UPX 加壳压缩）
输出的 GenP-v3.8.0.exe 放入 Release/ 目录
四、安全性分析
⚠️ 高风险/高关切点
1. 修改系统 DLL (wintrust.dll)

patch_wintrust.ps1 修改的是 Windows 的数字签名验证核心 DLL。wintrust.dll 负责验证文件的 Authenticode 数字签名。补丁将签名验证函数的返回值改为始终返回 0（成功），这会直接削弱系统的安全签名验证体系，可能导致：

无法正确验证任何程序的数字签名
系统更容易被恶意软件利用
2. 要求管理员权限

.au3 脚本开头包含 #RequireAdmin，运行时要求完整的管理员权限，能够：

修改系统 hosts 文件（屏蔽 Adobe 验证域名）
创建/修改 Windows 防火墙规则
修改 Program Files 下的 Adobe 二进制文件
篡改注册表
3. 二进制补丁行为

config.ini 中的 [Patches] 节包含了大量十六进制搜索/替换模式，直接修改 Adobe 程序的机器码：

绕过许可证验证函数（ValidateLicense）
跳过过期检查（ProfileExpired）
绕过签名完整性检查（AcroTegrity）
隐藏横幅广告（Banner）
绕过"盗版检测"机制（RunningVulture）
4. 网络请求

程序会从外部 URL https://a.dove.isdumb.one/list.txt 下载域名屏蔽列表，写入系统 hosts 文件。该 URL 内容不可控。

5. UPX 加壳

最终产物使用 UPX 压缩，这会导致很多杀毒软件报毒（无论实际是否有害），也使得逆向分析更困难。

从源码角度看是否"安全"（无恶意后门）
从我阅读的完整源码来看：

没有发现明显的后门、远程控制、数据窃取、键盘记录等恶意代码
没有发现除 Adobe 破解功能以外的隐藏功能
构建依赖（AutoIt、SciTE）从官方网站（autoitscript.com）下载
代码逻辑与其声称的功能一致
五、总结
维度	结论
语言	AutoIt v3 脚本 + PowerShell 构建辅助
编译	AutoIt3Wrapper 编译为 .exe + UPX 加壳
是否有恶意后门	源码中未发现后门或木马行为
是否安全	不安全 — 它会修改系统核心 DLL (wintrust.dll)、直接篡改二进制文件、修改防火墙和 hosts 文件，严重削弱系统安全机制
合法性	这是 Adobe 软件破解工具，使用它构成对 Adobe 软件许可协议的违反，在大多数地区属于违法行为
简单来说：源码本身没有藏后门，但这个工具的功能本身就是不安全的 — 它破坏了 Windows 的签名验证机制，并直接修改商业软件的二进制文件来绕过付费。
